# Final Portfolio & Presentation Rubric

## GENAI 490 — Capstone: AI Solution Design

**Total: 200 Points** (150 Portfolio + 50 Presentation)

---

## Portfolio Rubric (150 Points)

### 1. Project Narrative (30 pts)

| Score Range | Criteria |
|---|---|
| **27–30 (Excellent)** | Compelling, cohesive story showing clear evolution of thinking from HW1 through final product. Reads as a unified document, not stapled homework. Insights are specific and evidence-backed. Reader understands the project's significance and the student's growth. |
| **21–26 (Proficient)** | Coherent narrative covering all project phases. Some sections stronger than others. Shows some evolution but may rely on summarizing homework rather than synthesizing. |
| **15–20 (Developing)** | Reads like homework summaries connected by transitions. Minimal evidence of evolved thinking. Gaps in the project story. |
| **0–14 (Insufficient)** | Fragmented, incomplete, or largely copy-pasted from prior assignments without revision. |

### 2. Technical Artifacts (30 pts)

| Component | Points | Excellent | Proficient | Developing | Insufficient |
|---|---|---|---|---|---|
| Architecture diagram | 8 | Professional, updated, accurately reflects final system | Clear and labeled | Incomplete or outdated | Missing or inaccurate |
| Working demo | 10 | Polished demo of core workflow; handles edge cases gracefully | Core workflow works; minor issues | Partially working | Non-functional or missing |
| Code repository | 8 | Clean, documented, professional README, tests pass | Code works; README adequate | Messy code; minimal docs | No repo or broken code |
| Prompt library | 4 | Versioned prompts with evolution notes | Prompts included | Minimal prompt documentation | Missing |

### 3. Evidence Package (30 pts)

| Component | Points | Excellent | Proficient | Developing | Insufficient |
|---|---|---|---|---|---|
| User testing report | 10 | Rigorous methodology; 5+ participants; rich findings; AI trust analysis | 5 participants; adequate findings | Thin data; fewer than 5 | Missing or fabricated |
| Evaluation results | 8 | 15+ diverse test cases; insightful failure analysis; before/after comparison | 15 cases; basic analysis | Fewer than 15 or shallow | Missing or trivial cases |
| Business case | 7 | Realistic financials; math shown; assumptions sourced; sensitivity analysis | Numbers present; some assumptions weak | Incomplete or unrealistic | Missing or pure speculation |
| Success criteria comparison | 5 | Honest comparison with specific metrics; explains gaps | Comparison present | Vague or incomplete | Missing |

### 4. Reflection Essay (30 pts)

| Score Range | Criteria |
|---|---|
| **27–30 (Excellent)** | Deep, honest, specific reflection. Technical lessons tied to concrete experiences. Ethical thinking shows genuine evolution (not just "I learned ethics is important"). Career insights are personal and authentic. Advice to future students is practical and hard-won. |
| **21–26 (Proficient)** | Genuine reflection with some depth. Covers all required areas. Some insights are specific; others remain general. |
| **15–20 (Developing)** | Surface-level reflection. Mostly positive without meaningful self-critique. Generic lessons ("I learned to plan better"). |
| **0–14 (Insufficient)** | Missing, perfunctory, or clearly AI-generated without personal voice. |

### 5. Completeness & Professionalism (30 pts)

| Component | Points | Criteria |
|---|---|---|
| All appendices present | 8 | All 6 revised homework deliverables, raw data, cost model, prompt log |
| Writing quality | 8 | Professional prose; clear; grammatically correct; appropriate for employer audience |
| Formatting & structure | 7 | Consistent formatting; proper headings; page numbers; high-res diagrams |
| Citations & transparency | 7 | All claims sourced; AI tool usage documented; no unsupported assertions |

---

## Presentation Rubric (50 Points)

### 6. Content & Structure (20 pts)

| Score Range | Criteria |
|---|---|
| **18–20 (Excellent)** | Clear narrative arc; opens with compelling hook; demo is smooth and shows core value; technical depth appropriate for audience; evidence presented convincingly; honest about limitations. |
| **14–17 (Proficient)** | Covers all required sections; adequate demo; reasonable technical depth; some sections stronger than others. |
| **10–13 (Developing)** | Missing sections; demo issues; uneven depth; over-reliance on slides vs. showing the product. |
| **0–9 (Insufficient)** | Major gaps; no demo; reads slides verbatim; fails to communicate project value. |

### 7. Delivery & Presence (15 pts)

| Score Range | Criteria |
|---|---|
| **13–15 (Excellent)** | Confident, natural delivery; maintains eye contact; speaks to audience (not slides); handles transitions smoothly; stays within time limit. |
| **10–12 (Proficient)** | Generally clear delivery; occasional reliance on notes; mostly within time. |
| **7–9 (Developing)** | Reads from slides/notes frequently; rushed or significantly over time; nervous but functional. |
| **0–6 (Insufficient)** | Unprepared; cannot explain own project; significantly over/under time. |

### 8. Q&A Handling (15 pts)

| Score Range | Criteria |
|---|---|
| **13–15 (Excellent)** | Answers technical, business, and ethical questions with depth and honesty; admits knowledge gaps gracefully; connects answers to evidence from the project. |
| **10–12 (Proficient)** | Handles most questions adequately; occasionally vague or defensive. |
| **7–9 (Developing)** | Struggles with questions outside the presentation content; gives generic answers. |
| **0–6 (Insufficient)** | Cannot answer basic questions about own project; dismissive or evasive. |

## Grade Scale

| Points | Grade | Description |
|---|---|---|
| 180–200 | A | Exceptional capstone work; portfolio-ready for professional use |
| 160–179 | A- | Strong work with minor gaps |
| 140–159 | B+ | Solid work; some areas need more depth |
| 120–139 | B | Adequate work; meets requirements but lacks distinction |
| 100–119 | B- | Below expectations in several areas |
| 80–99 | C | Significant gaps; barely meets minimum requirements |
| <80 | F | Does not demonstrate capstone-level competency |

---

## Common Deductions

| Issue | Deduction |
|---|---|
| Late submission (per day) | -10 pts |
| Missing demo (no live or recorded) | -20 pts |
| No GitHub repository | -15 pts |
| Fabricated user testing data | -50 pts + academic integrity referral |
| Presentation over 20 minutes | -5 pts per additional minute |
| No backup demo (live demo fails) | -5 pts |
| Missing appendices (per item) | -3 pts |
| No AI tool usage transparency | -10 pts |
