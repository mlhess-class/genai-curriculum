# Homework 3: Prototype Report (Sprint 1 MVP)

## GENAI 490 — Capstone: AI Solution Design

**Milestone Deliverable 3 of 6** | **Due: Week 8** | **100 Points**

---

## Overview

Build and demonstrate a working MVP of your generative AI solution. This is Sprint 1 — focus on the core value proposition, not polish. The deliverable is both the working prototype and a written report documenting your build process.

## Learning Objectives

1. Implement a functional generative AI prototype end-to-end
2. Apply prompt engineering and/or RAG techniques in a real system
3. Document technical decisions made during implementation
4. Conduct basic evaluation of model outputs
5. Demonstrate iterative development practices

## Instructions

### Part 1: Prototype Demo (30 pts)

Provide access to your working prototype via **one** of:

- Live deployment URL (preferred)
- Screen recording (5–10 min) walking through core functionality
- GitHub repository with clear setup instructions and a recorded demo

Your prototype must demonstrate:

- [ ] Core AI functionality working end-to-end
- [ ] At least one complete user workflow
- [ ] Real (not mocked) model API calls
- [ ] Basic error handling

### Part 2: Technical Implementation Report (30 pts)

Document your build process:

1. **Technology stack** — What you actually used (vs. what you planned in HW2) and why anything changed
2. **Model integration details**
   - Which models/APIs and how they're called
   - Prompt templates (include actual prompts used)
   - RAG pipeline details if applicable (chunking strategy, embedding model, retrieval approach)
3. **Key implementation challenges** — At least 3 challenges you faced and how you resolved them
4. **Code architecture** — Brief overview of codebase structure (include a tree or diagram)

### Part 3: Output Evaluation (25 pts)

Evaluate your prototype's AI outputs:

- **Test cases** — Design and run at least 10 representative test cases
- **Evaluation criteria** — Define what "good output" means for your use case (accuracy, relevance, tone, format, etc.)
- **Results summary** — Table showing test inputs, expected outputs, actual outputs, and pass/fail assessment
- **Failure analysis** — For any failures, analyze root cause and propose fixes

### Part 4: Sprint Retrospective (15 pts)

Write a candid retrospective:

- **What worked well** — Techniques, tools, or approaches that paid off
- **What didn't work** — Dead ends, wrong assumptions, wasted effort
- **Deviations from architecture** — How does the actual build differ from HW2? Why?
- **Sprint 2 plan** — Top 3–5 priorities for the next iteration based on what you learned

## Submission Requirements

- **Format:** PDF or Markdown report (5–8 pages) + prototype access
- **Code:** GitHub repository link (public or shared with instructor)
- **Demo:** Either live URL or screen recording
- **Test results:** Include raw evaluation data as an appendix

| Criterion | Excellent (90-100%) | Proficient (70-89%) | Developing (50-69%) | Insufficient (<50%) |

| **Prototype Demo (30)** | Fully functional core workflow; clean UX; real AI integration | Core workflow works with minor issues | Partially working; significant gaps | Non-functional or mocked AI |
| **Technical Report (30)** | Thorough documentation; honest about changes from HW2; clear code architecture | Adequate documentation; some gaps | Sparse documentation; missing key details | Missing or superficial |
| **Output Evaluation (25)** | 10+ diverse test cases; clear criteria; insightful failure analysis | 10 test cases; basic evaluation | Fewer than 10 or weak evaluation criteria | Missing or perfunctory |
