# Homework 1: Workflow Thinking

**GENAI 201: AI-Powered Workflows & Automation**
**Due:** End of Week 2 | **Points:** 100

## Learning Objectives

- Analyze a manual business process end-to-end
- Identify bottlenecks, repetitive steps, and automation opportunities
- Create clear process documentation using flowchart notation
- Prioritize automation candidates by impact and feasibility

## Instructions

### Part A: Process Mapping (40 pts)

Choose a real manual business process you've encountered (or select from the list below). Document it thoroughly:

1. **Process Overview** (10 pts) — Write a 200–300 word description covering: who's involved, what triggers the process, what the output is, and how often it runs.

2. **Step-by-Step Map** (20 pts) — Create a flowchart or numbered step list that includes:
   - Every human action (label with 👤)
   - Every decision point (label with 🔀)
   - Every wait/delay (label with ⏳)
   - Every data transfer between systems (label with 🔄)
   - Minimum 10 steps

3. **Time Estimates** (10 pts) — For each step, estimate the time it takes. Calculate total process duration and identify the longest steps.

**Suggested processes** (if you don't have your own):
- Employee onboarding (offer letter → first day)
- Customer support ticket lifecycle
- Invoice approval and payment
- Content publishing pipeline
- Event registration and follow-up

### Part B: Automation Analysis (35 pts)

Using your process map from Part A:

1. **Identify Automation Points** (15 pts) — Mark every step that could be partially or fully automated. For each, explain:
   - What tool/technology could automate it
   - Whether it's full automation or human-in-the-loop
   - Any risks or edge cases

2. **Impact Matrix** (10 pts) — Create a 2×2 matrix (High/Low Impact vs. High/Low Effort) and place each automation candidate in the appropriate quadrant.

3. **Recommended Automation Sequence** (10 pts) — Propose a phased implementation plan (Phase 1, 2, 3) starting with quick wins.

### Part C: Reflection (25 pts)

Write 300–500 words addressing:

1. Which steps **should not** be automated, and why? (10 pts)
2. How would you measure success after automating this process? Name 3 specific metrics. (10 pts)
3. What could go wrong if automation is implemented poorly? (5 pts)

## Submission

- Single PDF or Markdown file
- Flowcharts may be created with any tool (draw.io, Miro, Excalidraw, hand-drawn and photographed)
- Include your name and the process you chose at the top
