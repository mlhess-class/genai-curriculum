How We Built the GenAI Certificate Program Using AI

Prepared for the Chief Information Officer · April 27, 2026 · Jack Hess, University of Michigan

Executive Summary

In April 2026, we used AI-powered development tools to design and produce a complete, deployment-ready 15-credit Graduate Certificate in Generative AI & Intelligent Automation — from initial concept to a live GitHub Pages site with full course materials, in approximately one week.

The project demonstrates how AI can dramatically accelerate academic program development while maintaining quality standards sufficient for dean-level approval.

The completed program is live at: https://mlhess-class.github.io/genai-curriculum/

What Was Built

The AI-assisted process produced 459 files comprising a complete academic program package:

How AI Was Used

The Technology

The program was developed using Claude (Anthropic's AI assistant) operating through OpenClaw, an AI agent framework providing persistent workspace and file management, multi-agent orchestration, iterative refinement through conversation, and automated deployment via GitHub Actions.

The Process

1
Research Phase AI synthesized the current landscape of GenAI education across institutions, producing a publishable-quality research paper grounding the curriculum in literature and institutional analysis.
2
Curriculum Architecture Program learning outcomes, course sequencing, prerequisite chains, and credit structure were designed through iterative human-AI conversation, with the human providing strategic direction and the AI executing detailed design.
3
Course Development Each of the five courses was developed with week-by-week syllabi, required readings and resources (real, current publications), assessment frameworks, and tool/technology specifications.
4
Assessment Creation 30 homework assignments with full answer keys, plus exams, final projects, and rubrics — all aligned to course learning objectives.
5
Multi-Perspective Review AI generated authentic expert reviews from four distinct professional viewpoints, each with substantive critiques and actionable recommendations. Student feedback profiles stress-tested the curriculum from learner perspectives.
6
Iterative Revision The dean's review process (preliminary → revisions → final approval) modeled real academic governance, with the AI both generating the critique and responding to it with curriculum improvements.
7
Web Production All HTML sites, reveal.js slide decks, and the program marketing page were generated and deployed to GitHub Pages via CI/CD.

Implications for the Institution

  1. Speed of program development — AI-assisted curriculum design can compress 12–18 months of traditional development into days, enabling the institution to respond to rapidly evolving fields before the content becomes obsolete.
  2. Quality floor, not ceiling — AI produces first drafts of professional quality that experts then refine, rather than replacing expert judgment. The multi-review process ensures human oversight at every decision point.
  3. Cost efficiency — A program package of this scope would traditionally require multiple faculty, instructional designers, and web developers over multiple semesters. The AI-assisted approach required one faculty member working with AI tools over approximately one week.
  4. Iterative improvement built in — Because the materials are generated programmatically and stored in version control, updating for new tools, frameworks, or pedagogical feedback is straightforward rather than requiring wholesale redesign.
  5. Reproducible methodology — This same process can be applied to other certificate programs, course redesigns, or institutional initiatives where rapid, high-quality content development is needed.
